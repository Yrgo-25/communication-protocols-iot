# Bilaga B

## RS-485
**RS‑485** är en elektrisk standard för seriell kommunikation som ofta används i industriella och inbyggda system.

Standarden definierar **hur signaler överförs på ledningar**, men inte själva protokollet. Det innebär att RS‑485 ofta används tillsammans med ett egenutvecklat protokoll eller etablerade protokoll som t.ex.:
* Modbus.
* Profibus.
* Egna industriella protokoll.

RS‑485 är populärt eftersom det:
* Klarar långa kablar.
* Fungerar bra i störiga miljöer.
* Kan koppla ihop många enheter på samma buss.

---

### Differential signalering
RS‑485 använder differential signalering. Det betyder att två ledningar används:
* **A**
* **B**

Signalen bestäms av spänningsskillnaden mellan A och B.

    A ----\
           >---- mottagare
    B ----/

Fördelar:
* Hög störimmunitet.
* Bättre signalintegritet.
* Längre kabelavstånd (ofta över hundratals meter).

---

### Bussstruktur
RS‑485 används ofta som en buss, där flera enheter delar samma kablar:

```text
  Master
    |
    +----- Device 1
    |
    +----- Device 2
    |
    +----- Device 3
```

Vanliga egenskaper:
* Upp till 32 enheter på en buss (klassisk specifikation).
* Alla enheter delar samma två signalledningar.
* Vanligtvis används en master och flera slavar.

---

# Terminering
För att undvika signalreflektioner används termineringsresistorer i ändarna av bussen.

    120Ω                         120Ω
     |                             |
     +---- Device ---- Device ----+

Termineringsresistorerna:
* Placeras endast i bussens ändar.
* Sätts normalt till `120 Ω` för att matcha kabelns impedans.

---

### Half-duplex och full-duplex
RS‑485 kan användas i två lägen.

#### Half‑duplex
Alla enheter delar samma två ledningar, men endast en enhet får sända åt gången:

```text
A/B  <---->  kommunikation i båda riktningar
```

Detta är den vanligaste konfigurationen.

---

#### Full‑duplex
Två separata ledningspar används.

```text
TX+ TX-
RX+ RX-
```

Detta möjliggör samtidig sändning och mottagning, men används mer sällan.

---

### Typisk kommunikationsmodell
I många system används en master/slave-modell:
* Mastern:
    * Initierar kommunikation.
    * Skickar förfrågningar.
* Slavarna:
    * Svarar på förfrågningar.
    * Sänder endast när de blir tillfrågade.

Exempel:

```text
Master -> Sensor 1: Status?
Sensor 1 -> Master: OK

Master -> Sensor 2: Status?
Sensor 2 -> Master: OK
```

Detta förhindrar kollisioner på bussen.

### Exempel på en enkel frame
Ett enkelt RS‑485-protokoll kan använda frames som t.ex.:

```text
+-----------+-----------+---------+-----------+
|    SOF    | Address   | Data    | Checksum  |
+-----------+-----------+---------+-----------+
```

Exempel:

```text
0xAA  0x02  0x37  0xD4
```

Betydelse:

| Fält     | Betydelse                 |
| -------- | ------------------------- |
| SOF      | Markerar början på framen |
| Address  | Mottagarens adress        |
| Data     | Payload                   |
| Checksum | Felkontroll               |


---

### RS‑485 i mikrokontrollers
RS‑485 används ofta tillsammans med UART:
* Mikrokontrollern skickar seriell data via UART.
* En RS‑485‑transceiver konverterar signalen till differential form.

Typisk koppling:

```text
    MCU UART TX/RX
           |
       RS‑485 Transceiver
           |
          A/B
           |
         BUSS
```

### Representation i mjukvara
RS-485 definierar det elektriska lagret, inte själva protokollet.  
I mjukvara representerar man därför oftast en frame i sitt eget protokoll som skickas över RS‑485.

En enkel representation kan se ut så här:

``` cpp
namespace rs485
{
struct Frame
{
    static constexpr std::size_t DataLen{16U};
    std::uint8_t data[DataLen]{};
    std::uint8_t sof{};
    std::uint8_t address{};
    std::uint8_t length{};
    std::uint8_t checksum{};
};
} // namespace rs485
```

| Fält     | Betydelse             |
| -------- | --------------------- |
| sof      | Start Of Frame        |
| address  | Mottagarens adress    |
| length   | Antal bytes i payload |
| data     | Payload               |
| checksum | Felkontroll           |

#### Exempel - Skapande av frame
I nedanstående exempel skapas en frame för att skicka `0x37` till nod 2 på bussen:

``` cpp
rs485::Frame frame{};
frame.sof     = 0xAA;
frame.address = 0x02;

frame.length   = 1U;
frame.data[0]  = 0x37;
frame.checksum = 0xD4;
```

---

#### Exempel - RS485-nod
En nod, är implementerad via en strukt `rs485::Node`, kan kontrollera om meddelandet är adresserat till den genom att matcha mottagaradressen med sin egen adress:

```cpp
namespace rs485
{
struct Node
{
    std::uint8_t address{};

    void onReceive(const Frame& frame) noexcept
    {
        if (address == frame.address)
        {
            // Handle frame.
        }
    }
};
} // namespace rs485
```

---

#### Exempel - Bussdistribution
I en simulering kan en buss, här implementerad via en strukt `rs485::Bus`, skicka en given frame till alla anslutna noder.  
Varje nod avgör sedan själv om den ska behandla eller ignorera meddelandet:

``` cpp
namespace rs485
{
struct Bus
{
    static constexpr std::size_t MaxNodes{32U};
    Node* nodes[MaxNodes]{};
    std::uint8_t nodeCount{};

    void transmit(const Frame& frame) noexcept
    {
        for (std::uint8_t i{}; i < nodeCount; ++i)
        {
            if (nullptr != nodes[i]) 
            { 
                nodes[i]->onReceive(frame); 
            }
        }
    }
};
} // namespace rs485
```

### Sammanfattning
Viktiga egenskaper hos RS‑485:
* Differential signalering.
* Hög störimmunitet.
* Långa kabellängder.
* Flera enheter på samma buss.
* Används ofta tillsammans med UART.

RS‑485 är därför mycket vanligt i industriella kommunikationssystem och inbyggda styrsystem.
